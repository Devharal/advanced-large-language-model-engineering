# Paper Notes

One file per must-read paper, prefixed with the module number. Each note should
be a short (1 page max) summary in your own words: problem, method, key result,
why it matters for the module's project. Do not paste abstracts verbatim.

## Naming Convention
`<module>-<short-slug>.md`, e.g.:
- `01-attention-is-all-you-need.md`
- `01-flashattention-2.md`
- `03-lora.md`
- `03-qlora.md`
- `04-graphrag.md`
- `06-ragas.md`
- `11-chinchilla-scaling-laws.md`
- `12-constitutional-ai.md`

## Suggested Note Template
```markdown
# <Paper Title> (<Authors>, <Year>)

**Module:** 0X
**Link:** <arxiv/doi link>

## Problem
What gap does this paper address?

## Method
Core idea in 3-5 sentences. Equations only if essential.

## Key Result
The headline number/finding.

## Relevance to Core Engineering Project
How this paper's method maps to something you implement.
```
